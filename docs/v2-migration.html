Migration Guide from v1 to v2

Please notice that our REST API v2 is backward-incompatible with REST API v1. Please see the changes and new features before migrating to v2.

What's Changed

  • The API implements the JSON API. The format and structure of both request and response for all of API resources changed in a backward-incompatible way.
  • GET /v1/users is now GET /v2/users/me
  • /v1/accounts is now /v2/credentials
  • /v1/tasks is now /v2/flows
  • /v1/components and /v1/repos are merged into /v2/components
  • /v1/exec/schedule has been split into:
    • /v2/components/{COMPONENT_ID}/versions/{GIT_HASH}/verify-credential
    • /v2/components/{COMPONENT_ID}/versions/{GIT_HASH}/dynamic-metadata
    • /v2/components/{COMPONENT_ID}/versions/{GIT_HASH}/select-model

What's New

  • /v2/organizations resource for managing organizations
  • /v2/lookups resource for managing lookup tables
